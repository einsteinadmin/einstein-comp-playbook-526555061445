When to run a cycle

Default cadence is annual (May), per the philosophy doc. Don't run more often than that without a real trigger — restlessness isn't a reason.

Standing rhythm

  • Annual mover comp review — kicks off mid-April, lands first Saturday of May or first Saturday after Roundtable alignment.
  • Quarterly manager comp review — separate cycle, runs alongside QGR.
  • Variable pay calibration — quarterly, owned by Matisen (performance matrix in EMMA).

Mid-cycle triggers (when to run early)

Cost-of-labor drift

BMs report multiple candidates walking at current rates. ClearCo applicant flow drops materially while retention holds. Competitor rates surface in screens.

Retention red flag

Lead-tier voluntary turnover spikes in a region. eNPS pay-themed comments cluster. A specific mover everyone wanted to keep walks for $1/hr.

Margin headroom unlocks

Variable pricing pulls revenue meaningfully above forecast. AGM % of revenue gives 2-3 percentage points of comp-to-AGM headroom that wasn't there at last review.

Structural change

New role added (e.g. Part-Time Lead). Position retired (e.g. Hall of Fame as pay band). Tiered markets opened (e.g. Houston/SA below ATX/DFW).

Pre-flight check (do these before announcing a cycle)

  1. Re-read the Compensation Philosophy doc. Has anything in it changed since last cycle?
  2. Pull last cycle's modeler state. What did we ship? What did we hedge on that's now worth revisiting?
  3. Confirm 6-week runway. If you can't get from kickoff to Saturday in 6 weeks, the cycle gets messy.
  4. Ben (or current VCFO) has bandwidth for a 90-min affordability session in week 1.

Pitfalls + lessons learned

Real ones from prior cycles. Each one we hit at least once. Add to this list every cycle.

Rate flips on Saturday — always

Pay-period boundary is Saturday. Mid-period rate changes break Rippling, confuse movers, create a paycheck that doesn't reconcile. If you can't land Saturday, push to next Saturday.

Audit the pay ladder in three places

The ladder appears in the stair-step display, the JS POSITIONS data, and the calculator's "promotion gate" output. They have to agree. Run a general-purpose audit agent against both announcement pages before deploy.

Tier $ offset math is mechanical — make it explicit

If you have Premium / Standard tiered markets, every position base in Standard = Premium minus exactly $X/hr. Bonus structure, PTO, benefits identical. State the rule in the file as a code comment. Audit the offset against all visible figures + JS data.

Monday create_update body — no CDATA wrap

Wrapping the body in CDATA renders the update as blank text. Use raw HTML. For @mentions use the mentionsList parameter, never inline HTML links.

Promise framing on take-home pay

"Take-home" implies after-tax. Use "Weekly gross pay" or "Weekly pay" on calculator outputs. Movers will assume after-tax otherwise and feel underpaid when their first check lands.

Full-Time Only benefits get an explicit tag

Parental leave, bereavement, health/dental/vision are full-time only at Einstein. Tag them visually on benefits cards. Don't say "(part-time hours count too)" on benefits that don't actually cover PT — it's worse than no information.

Don't share the modeler URL with BMs in pressure tests

Modeler exposes math + scenarios + open questions that aren't useful frontline noise. Share a clean 1-pager. Cameron + Paul + VCFO + Anne are the only roles with modeler access.

HOF reframe is harder than the math

If you retire a paid position (e.g. Hall of Fame), the affected cohort hears "you're getting paid less" first. Lead with the grandfathering structure. Schedule 1:1s in the first week. Frame the new state as honor-only ("award", "prestige") not as comp loss.

Don't surface the leadership modeler in public pages

The mover announcement pages must NOT link to the modeler. Anyone with the modeler URL sees aggregate comp math, performance distribution, and AGM math. Strip any "Numbers sourced from..." footer references before deploy.
